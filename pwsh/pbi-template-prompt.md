
# PBI Template (Prompt Reference)

Use this template to create a new PBI in `docs/backlog` for Copilot. Fill in each section below and use checkboxes for acceptance criteria.

## Title
Short, action-oriented summary of the feature, fix, or improvement.

## Summary
Brief description of the problem, opportunity, or goal. State what will change and why.

## Acceptance Criteria
- [ ] List clear, testable requirements for completion
- [ ] Use checkboxes for each criterion
- [ ] Include UI, logic, error handling, and test coverage as needed

## Tasks
1. List implementation steps (code, UI, tests, docs)
2. Use numbered list for clarity

## Risks & Edge Cases
- List any known risks, edge cases, or blockers
- Note mitigation strategies if relevant

## Test Plan
- Describe how the feature/fix will be tested
- Include unit, integration, and manual/QA steps as needed

## Effort Estimate
Small/Medium/Large: X–Y dev days (or hours)

---

**Prompt Pattern:**
- Reference this template when creating a new PBI for Copilot.
- Fill in each section with specific details for your feature or fix.
- Use checkboxes and lists for clarity and traceability.
